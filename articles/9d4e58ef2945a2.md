## はじめに

現在、ターミナルには [ghostty](https://ghostty.org/)、エディタには neovim を使っています。
Claude Code を筆頭にターミナル上で AI と対話する機会が増え、neovim と AI セッションを頻繁に行き来するようになりました。

ghostty はタブ生成とペイン分割に対応しているので、一見これだけで十分そうですが、理想とする構成には1つレイヤーが足りませんでした。

## 理想のレイアウト

私の理想形なレイアウトはこちらです。

```
ghostty
├── Frontend
│   ├── neovim
│   ├── Claude Code
│   └── Web Server
└── Backend
    ├── neovim
    ├── Claude Code
    └── API Server
```

プロジェクト単位でディレクトリを固定したセッションをまとめる構成です。
ghostty のペイン分割でも実現はできます。ただ、neovim 内でもバッファ分割を頻繁に行うため、ターミナル側でもペインを分割してしまうとモニターの表示領域を大きく奪ってしまいます。
一度に視認できる範囲が狭まり、作業効率が落ちてしまうのが悩みでした。

## tmux を導入してみる

そこで導入したのが tmux です。
**ghostty タブ = tmux セッション** とし、その中で複数のウィンドウを管理することで理想の構成を実現していました。

普段の運用では困っていなかったのですが、次の点が引っかかっていました。

- tmux の知見が必要で、ハードルが高い
- ghostty の タブ/Pane と tmux の session/Window が概念として重複していて無駄が多い
- tmux のキーバインドやプレフィックスが干渉するため、`tmux.conf` で必要なキーのみ有効にするなど、設定の複雑さが増す

## cmux との出会い

そんなときに見つけたのが **cmux** です。

cmux のヒエラルキーは以下になっています。

```
Window
  └── Workspace (sidebar entry)
        └── Pane (split region)
              └── Surface (tab within pane)
                    └── Panel (terminal or browser content)
```

この **Workspace** がまさに求めていたものです。
ghostty + tmux でやっていた 「プロジェクト単位でセッションを束ねる構成」 が、Workspace を使えばcmuxのみで実現できます。

## cmux に移行してみた

Homebrewでインストール

```sh
brew tap manaflow-ai/cmux
brew install --cask cmux
```

ghostty の設定がそのまま利用できるため、config の変更は不要です。

実際に移行してみたレイアウトがこちら。

![cmuxレイアウト](/images/articles/cmux_layout.png)

cmux だけで理想のレイアウトが構築できました。tmux は不要になり、管理するものが一つ減ってスッキリしました。

さらに cmux には **内蔵ブラウザ** があり、Web サーバーの確認を同じアプリ内で完結できる点も気に入っています。

![cmuxブラウザ](/images/articles/cmux_browser.png)

:::message
内蔵ブラウザはちょっとした動作確認向けや調査に使うものとして、
最終的な成果物の確認は想定ブラウザで行うことを個人的にはおすすめします。
:::

## まだ歯がゆいところ

とはいえ、まだ改善を期待している点もあります。

- **設定のファイル管理ができない**
  ghostty は config ファイルで設定を管理できますが、cmux 固有の設定も同様にファイルから読み込めるようになってほしいです。
- **ヘッダーに全 Workspace を表示してほしい**
  私の運用ではサイドバーは基本的に非表示にしているため、ghostty のようにヘッダーにタブ形式で表示できたより良いです。
- **レイアウト・状態の復元は tmux ほどではない**
  ただし、終了時点のレイアウトやタブ名は復元してくれるため、実用上は許容範囲内かなと思っています。

## まとめ

cmux を導入することで、tmux なしで理想のワークスペース構成が実現できました。ghostty の設定がそのまま使えるので移行コストも低く、すんなり乗り換えられました。

ターミナル上で AI と並走する作業が増えている方には、ぜひ試してみてほしいツールです。
